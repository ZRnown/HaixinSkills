# warm-real-story-collection — 暖心人间故事搜集

## 技能定位

搜集/创作「收纸箱的妈妈」类型的短篇暖心故事 —— 普通人面对困境时的尊严、邻里间不声张的善意。不煽情、不说教、用具体的细节说话。

## 故事原型

```
一条街/一个小区/一个群里，住着一个正在经历困难的人。
他/她做了一件很小的事——在群里发了一段话、摆了一个摊、开了一个小店。
这件事里藏着一句话："我想靠自己。"
邻居们看到了，什么也没说，但都默默做了一件事。
后来，那件小事变成了一个不成文的规矩。
```

## 故事特征清单（判断标准）

- 主人公有明确的困境（单亲、残疾、年老、生病、贫穷），但不是卖惨
- 主人公有明确的自尊线："靠自己"，不是伸手要
- 邻里/陌生人的回应是「行动」而非「言语」——不声张、不宣扬
- 有一个具体可感的细节（纸箱攒在门口、纸条写"不急"、板子裱起来）
- 结尾安静、不过度拔高，读者自己品出暖意
- 篇幅 200-400 字，一段到底或两段，语言朴素

## 搜集流程

### 第 1 步：多源并行搜索

每次至少搜 3 个方向，每个方向用不同关键词组合：

**方向 A — 知乎（长文社区，故事质量高）**
```
知乎 + "邻居" + "暖心/感动/真实"
知乎 + "单亲妈妈/保洁阿姨/外卖/修鞋" + "靠自己"
知乎 + "业主群" + "默默"
知乎 + "你见过最温暖的" + 邻里/陌生人
```

**方向 B — 小红书（生活社区，细节丰富）**
```
小红书 + "真实故事" + "邻居" + "暖心"
小红书 + "小区" + "业主群" + "感动"
小红书 + "纸箱/废品/打扫" + "妈妈/阿姨/老人"
```

**方向 C — 通用/新闻/微信**
```
"我想靠自己" + "孩子" + 邻居/小区
"门口" + "修鞋/缝纫/收废品" + 老人/妈妈 + 真实故事
保洁阿姨/门卫 + 小区 + 暖心
```

**方向 D — 英文素材（可改编）**
```
Reddit r/HumansBeingBros neighbor
Reddit r/MadeMeSmile single mom/dad neighbor
"neighbor quietly" kindness story
```

执行：`web_search` 并行发起，每方向 ≥2 次搜索，每次不同关键词。

### 第 2 步：筛选与溯源

命中后：
1. 用 `web_fetch` 打开原文，确认故事完整可用
2. 标注来源 URL、发布时间、作者（如有）
3. 判断：可直接引用 / 需改写 / 仅作灵感参考

筛选标准：
- 有具体的人 + 具体的事 + 具体的细节 → 收录
- 泛泛的 "很感动" "邻居真好" 无细节 → 丢弃
- 明显编造、营销号鸡汤 → 丢弃

### 第 3 步：改写/原创

**改写（有原始素材）**：
- 压缩到 200-400 字
- 保留最核心的 2-3 个具体细节
- 去掉形容词堆砌、感慨、道德总结
- 保持口语感，像在跟朋友讲一件事

**原创（无素材，直接生成）**：
按故事原型构建：
1. 定人物：谁 + 什么困境 + ta 在做什么
2. 定动作：ta 怎么表达"靠自己"（一句具体的请求/一个具体的做法）
3. 定回应：邻居们做了什么具体的事
4. 定收尾：一个安静的、可回味的细节

示例人物池：
- 小区里收纸箱的单亲妈妈
- 门口修鞋的残疾老人
- 楼道里摆缝纫机的下岗女工
- 深夜代驾的单亲父亲
- 菜市场角落里卖菜的老奶奶
- 快递驿站里带孩子的聋哑妈妈
- 小区门卫室帮人免费看孩子的退休大爷
- 工地旁摆小摊卖盒饭的夫妻

### 第 4 步：统一输出格式

```
## 故事标题（可选，一句话）

[故事正文，200-400 字，纯文字无格式]

---
来源：[URL 或 "原创"]
人物：[主人公特征]
困境：[简要]
核心动作：[ta 靠自己做什么]
邻里回应：[具体做了什么]
核心情绪：[尊严 / 温暖 / 被看见 / 重新被需要 / ...]
适用场景：[内容创作 / 亲子共读改编 / 短视频脚本 / ...]
```

## 故事库管理

搜集到的故事统一存入 `stories/` 目录，文件名：`YYYY-MM-DD_简短标题.md`。

每次搜集完成后更新 `stories/INDEX.md`（表格：标题 | 人物 | 情绪 | 来源 | 日期）。

## 注意事项

- 涉及真人的故事，如果来源是小红书/知乎等公开平台，引用时标注出处
- 原创故事不碰敏感话题（政治、宗教、社会矛盾）
- 保持「暖和但不烫」的调性——不煽情、不消费苦难、不制造焦虑
- 同一主题可做变体（换人物/换场景/换困境），但每次变体要有新的核心细节
